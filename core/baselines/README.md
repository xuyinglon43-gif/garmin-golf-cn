# 基线数据说明

每个 JSON 文件存储一个差点档位的"**期望杆数表**"——给定**球的位置**（落点类型 + 距离洞距离），从这个位置打到进洞，统计上平均还要多少杆。

## 数据结构

```json
{
  "name": "业余 15 差点",
  "label": "amateur_15",
  "source": "Mark Broadie + Lou Stagner + Arccos 公开统计估算",
  "version": "0.1.0",

  "tee": {
    "par4": { "300": 4.10, "350": 4.18, ... },
    "par5": { "470": 4.95, ... }
  },

  "fairway": {
    "20": 2.50, "40": 2.55, "60": 2.65, ...
  },

  "rough": {
    "20": 2.65, "40": 2.75, ...
  },

  "sand": {
    "20": 2.85, "40": 3.05, ...
  },

  "recovery": {
    "20": 3.20, ...
  },

  "green": {
    "1": 1.05, "3": 1.40, "5": 1.60, ...
  }
}
```

距离单位：**码**（yards）。`green` 段距离单位也是码（推杆距离）。

## Strokes Gained 计算原理

每一杆的 SG = 起点期望杆数 − 终点期望杆数 − 1

举例（摘自 Broadie）：
- 起点：球道，距洞 150 码 → 期望杆数 2.93
- 落点：果岭，距洞 5 码（约 15 英尺）→ 期望杆数 1.83
- 这一杆 SG = 2.93 − 1.83 − 1 = **+0.10 杆**（比基线赚 0.10 杆）

进洞时终点期望杆数 = 0。

## 数据来源与精度

### PGA Tour 基线（`pga_tour.json`）
Mark Broadie 在《Every Shot Counts》(2014) 第 9 章发布的官方表格，基于 ShotLink 系统千万级击球数据。这是 SG 框架的金标准。

### 业余球手基线（`amateur_*.json`）
**注意：业余基线没有像 PGA Tour 那样的大规模公开数据集**。我们的基线是基于以下来源拟合的：

1. Broadie 在《Every Shot Counts》业余章节发布的差点 0/9/15/18 抽样数据
2. Lou Stagner（[X: @LouStagner](https://x.com/LouStagner)）公开的业余分布研究
3. Arccos 公司发布的差点段平均统计
4. 简化的距离-杆数缩放模型

**精度估计**：±0.1 杆/位置。对于普通业余球手个人分析，这个精度足够找出主要弱点。后续版本会随着数据收集逐步精化。

如果你有更准确的业余球手数据来源，欢迎提 PR 或 issue。

## 差点档位

| 文件 | 差点 | 大致水平 |
|---|---|---|
| `pga_tour.json` | — | PGA Tour 平均（参考用，业余球手别拿这个比） |
| `amateur_scratch.json` | 0 | 单位数差点的业余顶尖 |
| `amateur_5.json` | 5 | 准单位数 |
| `amateur_10.json` | 10 | 中等业余 |
| `amateur_15.json` | 15 | 中等业余偏下 ⭐（许多球友的当前水平） |
| `amateur_20.json` | 20 | 周末玩家 |
| `amateur_25.json` | 25 | 入门 |

## 使用建议

**别用 PGA Tour 基线评估自己**——你 15 差点跟 Tour 比 SG 都是 -3 杆/轮，毫无意义。

**用同差点或临近差点档位评估**：
- 你 15 差点 → 用 `amateur_15.json` 或 `amateur_10.json`（如果想看追赶目标）
- "我比 15 差点平均球手在进攻杆少丢 0.3 杆/轮" → 这才是有意义的比较
